⚙ How the Dashboard Works

1 · Board List — Dynamically Fetched

On connect, the dashboard calls the Azure DevOps Classification Nodes API at depth 10, returning your full area path tree. Any new board added in future appears automatically.

GET /wit/classificationnodes/areas?$depth=10&api-version=7.1

2 · No 500-Ticket Limit

Standard WIQL caps at 500. This dashboard uses continuation tokens to page through all results up to your chosen max.

POST /wit/wiql?api-version=7.1 -- keep fetching while continuationToken exists: POST /wit/wiql?continuationToken={token}&api-version=7.1

3 · Board-Aware Categorisation

Tickets are categorised using title + description keyword rules. What makes this board-aware is that the category list is filtered to only show categories that actually appear in the data — so a Features board will only show feature-relevant categories, while a Maintenance board shows maintenance-specific ones.

Rules are checked top to bottom. First match wins. The full rule set:

CategoryKeywords matched in title + description

4 · Auto-Generated Insights

After categorisation the dashboard computes pattern findings from the actual numbers — highest volume, worst open rate, highest bug rate, manual ops burden, and resolution speed. No external API needed.

5 · KPI Definitions

  • Open — State not in: Closed, Resolved, Done
  • Bugs — Work Item Type contains Bug, Spider bug, or Defect
  • Manual Ops — Type is Spindling or Manual
  • Avg Days to Close — mean of ClosedDate − ActivatedDate (falls back to CreatedDate)
  • Pages Fetched — WIQL pagination calls made
